## 🎯 What is PostFlow?

PostFlow solves a real problem: **local service businesses don't have time for social media**. A plumber working 60 hours a week can't also be a content creator. PostFlow handles it for them.

Customers sign up, paste their website URL, and PostFlow generates a steady stream of professional, on-brand social media content — automatically scheduled and posted across all their platforms. The AI uses real data from their website (not generic templates), so the content actually reflects their business.

### The Market

- **$2.2B → $10.33B** social media management market (2024-2029, 36.2% CAGR)
- **750,000+** underserved local service businesses in the US alone
- Existing tools (SocialBee, Buffer) require manual content creation — PostFlow generates it

### The Differentiator

Unlike competitors that just *schedule* content you've already created, PostFlow:
1. **Generates content from scratch** using AI (text + images + carousels + videos)
2. **Scrapes your website** to ensure content reflects your actual services (not generic stock content)
3. **Costs ~$3/month** to operate per customer at scale (vs $99-349/month subscription) — **98%+ gross margins**

## 🔄 How It Works

```
┌─────────────────────────────────────────────────────────────┐
│                    CUSTOMER ONBOARDING                       │
│  1. Sign up (3-min flow: business → branding → AI avatar)   │
│  2. Paste website URL → FREE scrape extracts services       │
│  3. Receive 10 free credits (7-day trial)                   │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│                    CONTENT GENERATION                         │
│  Manual: User prompt → Claude (caption) +                    │
│          NanoBanana/Midjourney (image) → preview → schedule  │
│                                                               │
│  Auto: Cron job (2am daily) → 7-day plan via Claude →        │
│        Generate images for each post → Schedule              │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│                       POSTING                                 │
│  Cron job (every hour) checks for due posts →                │
│  Posts to FB/IG/GBP via OAuth →                              │
│  Tracks engagement (likes/comments/shares)                   │
└─────────────────────────────────────────────────────────────┘
```

### The Credit-Saving Flow

This is what makes PostFlow economical:

1. **Customer pastes website URL** → Scraper extracts services + about info (FREE)
2. **Data saves to `customers` table** → cached for 7 days
3. **AI generation** reads `website_services` from DB → injects into Claude prompt
4. **Claude knows real business context** → generates accurate content first try
5. **No wasted regenerations** → 50-70% fewer credits spent per usable post

**Result:** A Professional plan ($199/150 credits) yields ~225 usable posts instead of ~75.

## 🚀 Quick Start (Replit)

The fastest path to running PostFlow.

### 1. Upload to Replit

1. Go to [replit.com](https://replit.com) → "Create Repl" → "Import from upload"
2. Upload `postflow-complete.zip`
3. Replit auto-detects Node.js + PostgreSQL from `.replit` config

### 2. Provision Database

1. Click the **Database** tab in left sidebar
2. Click "Create database" → select PostgreSQL
3. Wait ~30 seconds for provisioning
4. `DATABASE_URL` is automatically set as a secret

### 3. Run Schema

In the Replit Shell:

```bash
psql $DATABASE_URL -f backend/db/schema.sql
```

You should see: `Schema created successfully`

### 4. Add API Keys (Secrets)

Click the **🔒 Secrets** tab and add:

**Required:**
| Key | Where to Get | Purpose |
|-----|--------------|---------|
| `ANTHROPIC_API_KEY` | [console.anthropic.com](https://console.anthropic.com) | Caption generation |
| `GOOGLE_AI_API_KEY` | [aistudio.google.com/app/apikey](https://aistudio.google.com/app/apikey) | NanoBanana images |
| `JWT_SECRET` | Run `openssl rand -hex 32` in shell | Auth signing |
| `CLOUDINARY_CLOUD_NAME` | [cloudinary.com](https://cloudinary.com) (free 25GB) | Image storage |
| `CLOUDINARY_API_KEY` | Same as above | Image storage |
| `CLOUDINARY_API_SECRET` | Same as above | Image storage |

**Optional:**
| Key | Purpose |
|-----|---------|
| `REPLICATE_API_TOKEN` | Midjourney fallback |
| `HEYGEN_API_KEY` | Video generation |
| `IMAGE_PROVIDER` | Set to `nanobanana` (default) or `midjourney` |

### 5. Install & Run

```bash
npm run install:all
```

Then click the green **Run** button. The app will:
- Start backend on port `3001`
- Start frontend on port `3000`
- Open the preview pane

### 6. Test It

1. Open the preview URL
2. Click **"Start free trial"**
3. Complete the 2-step signup
4. You're in! Generate your first AI post.

## 💻 Local Development

### Prerequisites

- **Node.js 18+** ([download](https://nodejs.org))
- **PostgreSQL 14+** ([download](https://www.postgresql.org/download/))
- **Git** (optional, for version control)

### Step-by-Step Setup

```bash
# 1. Clone or extract the project
unzip postflow-complete.zip
cd postflow

# 2. Install all dependencies (root + backend + frontend)
npm run install:all

# 3. Create the database
createdb socialmedia

# 4. Run the schema
psql -d socialmedia -f backend/db/schema.sql

# 5. Configure environment
cp backend/.env.example backend/.env
# Edit backend/.env with your API keys (see Environment Variables section)

# 6. Run development server
npm run dev
```

| `npm run build` | Build frontend for production |
| `npm run db:init` | Initialize the database schema |

## 📡 API Reference

All authenticated endpoints require a `Bearer` token in the `Authorization` header.

### Authentication

| Method | Endpoint | Description | Auth |
|--------|----------|-------------|------|
| `POST` | `/auth/register` | Create account (returns 10 free credits) | No |
| `POST` | `/auth/login` | Login → JWT token | No |
| `GET` | `/auth/verify` | Verify token, get current user | Yes |

**Example:**
```bash
curl -X POST http://localhost:3001/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "owner@plumbing.com",
    "password": "secure123",
    "businessName": "ABC Plumbing",
    "industry": "plumbing",
    "location": "Austin, TX"
  }'
```

## 🕷️ Website Scraping (Credit Saver)

This is PostFlow's secret weapon for keeping content costs low.

### How It Works

```
USER FLOW:
┌──────────────────────────────────────────────────────┐
│ 1. User signs up                                      │
│ 2. Goes to Settings → "Website Intelligence" section  │
│ 3. Pastes website URL → clicks "Scrape Site"          │
│ 4. ScraperService extracts:                           │
│    • Business name & description                      │
│    • Services offered                                 │
│    • About info                                       │
│    • Contact details                                  │
│    • Images & social links                            │
│ 5. Data saved to DB (cached 7 days)                   │
└──────────────────────────────────────────────────────┘

CONTENT GENERATION FLOW:
┌──────────────────────────────────────────────────────┐
│ 1. User clicks "Create Post"                          │
│ 2. Backend reads customer.website_services from DB    │
│ 3. Injects into Claude prompt:                        │
│    "Actual services: Driveway installation, Concrete  │
│     repair, Stamped concrete..."                      │
│ 4. Claude generates accurate, on-brand content        │
│ 5. User likes it → keeps it → 1 credit spent          │
│                                                       │
│ vs. without scraping:                                 │
│ → Generic content → user regenerates 2-3x             │
│ → 3-9 credits wasted per usable post                  │
└──────────────────────────────────────────────────────┘
```

### Credit Savings Math

| Scenario | Credits per usable post |
|----------|------------------------|
| Without scraping | 3-9 (multiple regenerations) |
| With scraping | 1 (first try works) |
| **Effective savings** | **50-70%** |

For a Professional plan customer (150 credits/month):
- Without scraping: ~75 usable posts
- With scraping: **~225 usable posts** (3x more value)

### Cache Strategy

- Scraped data is cached in `customers.scraped_data` for **7 days**
- Re-scraping same URL within 7 days returns cached data instantly
- User can clear cache anytime in Settings

## 🤖 AI Providers

PostFlow uses a multi-provider strategy for resilience and cost optimization.

### Image Generation: NanoBanana vs Midjourney

| Feature | 🍌 NanoBanana | 🎨 Midjourney |
|---------|---------------|----------------|
| **Cost per image** | ~$0.039 | ~$0.08 |
| **Speed** | 3-8 seconds | 15-20 seconds |
| **Quality** | Excellent | Premium artistic |
| **Editing support** | ✅ Conversational | ❌ |
| **Best for** | Daily posts, carousels | Hero/campaign content |
| **Default** | ✅ Yes | No |

### Smart Provider Selection

The `ManualContentGenerator` automatically picks the right provider:

```javascript
// Priority order:
1. Customer's `preferred_image_provider` (set in Settings)
2. Environment `IMAGE_PROVIDER` variable
3. First available provider with API key
4. Throws error if none configured
```

### Customer Override

Users can change their preferred provider in **Settings → Image Generation Provider**. The UI shows which providers are active (have API keys) and which aren't.

## 🗄️ Database Schema

PostFlow uses 7 PostgreSQL tables. See `backend/db/schema.sql` for the full DDL.

### `customers`
Business accounts with branding, credits, subscription, and scraped website data.

**Key columns:**
- `email`, `password_hash`, `business_name`, `industry`, `location`
- `brand_colors` (JSONB), `visual_style`, `tone`
- `avatar_id`, `voice_id`, `preferred_image_provider`
- `plan`, `credits_balance`, `credits_used_this_month`
- `scraped_data` (JSONB), `website_services` (JSONB), `website_about`

### `social_accounts`
OAuth-linked Facebook/Instagram/Google Business accounts.

**Key columns:**
- `customer_id` → `customers.id`
- `platform`, `access_token`, `refresh_token`
- `account_username`, `profile_image_url`
- `enabled`, `auto_post`

### `posts`
All generated/scheduled posts.

**Key columns:**
- `customer_id`, `content_type` (static/photo/carousel/video)
- `caption`, `media_url`, `hashtags` (JSONB)
- `prompt`, `image_provider`, `ai_model_used`
- `scheduled_date`, `posted_at`, `status`
- `engagement` (JSONB: likes, comments, shares)

### `post_carousel_slides`
Individual slides for carousel posts.

### `content_templates`
Reusable prompt templates per customer.

### `credit_transactions`
Full audit trail of credit debits and credits.

### `industry_templates`
Pre-built prompts for 5+ industries × 7 daily themes (35+ templates seeded).

## 🐛 Troubleshooting

### Database Connection Errors

```bash
# Check if PostgreSQL is running
pg_isready

# On Replit, verify DATABASE_URL is set
echo $DATABASE_URL

# Re-initialize schema if needed
psql $DATABASE_URL -f backend/db/schema.sql
```

### "Module not found" Errors

```bash
# Reinstall all dependencies cleanly
rm -rf node_modules backend/node_modules frontend/node_modules
npm run install:all
```

### Port Already in Use

```bash
# Kill processes on default ports
lsof -ti:3000 | xargs kill -9
lsof -ti:3001 | xargs kill -9
```

### NanoBanana Not Working

- Verify `GOOGLE_AI_API_KEY` is set in Secrets/`.env`
- Check key is valid at [aistudio.google.com](https://aistudio.google.com/app/apikey)
- The model name is `gemini-2.5-flash-image-preview` (preview status)

### Scraper Times Out

Some sites are slow or block scrapers. In `backend/services/ScraperService.js`:

```javascript
this.timeout = 30000;  // Increase from 15000
```

For sites with heavy JavaScript (Shopify, etc.), the regex-based scraper may not extract enough — consider adding Puppeteer for those cases.

### Content Generation Fails

Check the `/health` endpoint:

```bash
curl http://localhost:3001/health
```

It shows which AI services have valid API keys.
